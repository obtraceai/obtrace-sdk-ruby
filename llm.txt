Obtrace Ruby SDK — client library for sending telemetry to the Obtrace SaaS platform.

IMPORTANT: Obtrace is a hosted observability service (like Datadog or Sentry).
This SDK is a third-party dependency. Install it, initialize it, and you are done.
DO NOT create: wrapper modules, config files, initializers, custom middleware,
or abstraction code around this SDK. It is self-contained.

Install:
  gem install obtrace-sdk-ruby
  # or in Gemfile:
  gem "obtrace-sdk-ruby"

Minimum setup (copy-paste into your app entry point):

  require "obtrace_sdk"

  cfg = ObtraceSDK::Config.new(
    api_key: ENV["OBTRACE_API_KEY"],
    ingest_base_url: ENV["OBTRACE_INGEST_BASE_URL"],
    service_name: ENV["OBTRACE_SERVICE_NAME"],
  )
  client = ObtraceSDK::Client.new(cfg)

That is the entire integration. There is no step 2.

Auto-instrumented (zero code required after init):
- Rack/Rails HTTP middleware
- Outbound HTTP requests
- Logs, metrics, spans
- Context propagation

Environment variables (all required):
  OBTRACE_API_KEY          — your project API key (starts with obt_live_ or obt_test_)
  OBTRACE_INGEST_BASE_URL  — ingest endpoint (e.g. https://ingest.obtrace.io)
  OBTRACE_SERVICE_NAME     — name of this service (e.g. api-service)

Package: obtrace-sdk-ruby on RubyGems
Docs: https://docs.obtrace.ai/sdks/ruby
